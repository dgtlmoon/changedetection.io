{{ _('Accessibility statement') }}

{{ _('onChange by Sairo is committed to digital accessibility. We target conformance with the Web Content Accessibility Guidelines (WCAG) 2.2 Level AA across both the public marketing pages and the application itself.') }}

{{ _('Conformance status') }}

{{ _('This product is') }} {{ _('substantially conformant') }} {{ _('with WCAG 2.2 Level AA. "Substantially conformant" means the application meets the WCAG 2.2 Level AA success criteria with the limited, scoped exceptions documented in the VPAT and below.') }}

{{ _('A detailed, per-criterion conformance report is published as our') }} {{ _('Voluntary Product Accessibility Template (VPAT 2.5Rev — WCAG 2.2)') }}.

{{ _('Standards we measure against') }}

  • {{ _('WCAG 2.2 Level A and AA — primary target.') }}
  • {{ _('WAI-ARIA 1.2 Authoring Practices for custom widgets (dialogs, tabs, accordions, menus).') }}
  • {{ _('Revised Section 508 of the U.S. Rehabilitation Act (which incorporates WCAG 2.0 AA by reference; we exceed this baseline).') }}
  • {{ _('EN 301 549 v3.2.1 (the European harmonised accessibility standard for ICT, which references WCAG 2.1 AA; we exceed this baseline).') }}

{{ _('Compatibility') }}

{{ _('We test against the following user-agent and assistive-technology pairings before releasing accessibility-impacting changes:') }}

  • {{ _('macOS Safari + VoiceOver (latest stable)') }}
  • {{ _('iOS Safari + VoiceOver (latest stable)') }}
  • {{ _('Windows Firefox + NVDA (latest stable)') }}
  • {{ _('Windows Chrome + JAWS (latest two major versions)') }}
  • {{ _('Android Chrome + TalkBack (latest stable)') }}
  • {{ _('Linux Firefox + Orca (best-effort)') }}

{{ _('JavaScript is required for some interactive features (live diff preview, AI-assisted filter builder, real-time watch status). Static views — landing, features, about, privacy, login, watchlist — are usable without JavaScript.') }}

{{ _('How we get here') }}

  • {{ _('Every interactive control is reachable, visible, and operable via keyboard alone.') }}
  • {{ _('Focus indicators are a 2 px indigo ring with 2 px offset; they are never removed.') }}
  • {{ _('Sticky chrome (header, mobile bottom nav) is sized so the focused element is never fully obscured (WCAG 2.2 SC 2.4.11).') }}
  • {{ _('Touch targets are at minimum 24 × 24 CSS px globally and 44 × 44 CSS px on touch viewports (WCAG 2.2 SC 2.5.8 / 2.5.5).') }}
  • {{ _('Drag-and-drop interactions (re-ordering watches, resizing) all expose a keyboard or click-based equivalent (WCAG 2.2 SC 2.5.7).') }}
  • {{ _('Login uses a standard email + password flow with paste enabled and password-manager autofill supported; no cognitive-function tests are required (WCAG 2.2 SC 3.3.8).') }}
  • {{ _('The Help / Docs link appears in the same relative position on every page (WCAG 2.2 SC 3.2.6).') }}
  • {{ _('Multi-step flows pre-fill any value the user has already entered earlier in the same flow (WCAG 2.2 SC 3.3.7).') }}
  • {{ _('All text and meaningful UI components meet WCAG 2.2 contrast minima (4.5 : 1 normal text, 3 : 1 large text and non-text components). Each pair is logged in DESIGN.md § 2.4.') }}
  • {{ _('Animation respects prefers-reduced-motion. No content flashes more than three times per second.') }}

{{ _('Known limitations') }}

{{ _('We disclose the items below transparently. Each is tracked in our public issue tracker and we welcome additional reports.') }}

  • {{ _('Visual selector (browser-steps capture)') }} — {{ _('the in-browser CSS / XPath picker relies on pointer interaction with a rendered page screenshot. Direct keyboard equivalents are limited; users who cannot use a pointing device can paste a CSS selector or XPath into the same form field as a workaround. Improving this is on the roadmap.') }}
  • {{ _('Embedded third-party content') }} — {{ _('the page snapshots the application captures from sites you monitor are user-supplied content; their accessibility is outside our control. The diff viewer surrounding them is fully accessible.') }}
  • {{ _('Animated GIFs in documentation') }} — {{ _('a small number of demonstrational GIFs in the docs/ folder do not have synchronised captions; they always sit beside an equivalent textual description.') }}

{{ _('Assessment approach') }}

{{ _('Our conformance claim is supported by a combination of:') }}

  • {{ _('Automated testing with axe-core and Lighthouse in development.') }}
  • {{ _('Manual keyboard-only walkthroughs of every changed flow.') }}
  • {{ _('Manual screen-reader smoke-tests against at least one of VoiceOver / NVDA / Orca per accessibility-impacting change.') }}
  • {{ _('Reflow checks at 320 CSS px and 400% zoom.') }}
  • {{ _('Colour-blind simulation (protanopia, deuteranopia, tritanopia) for all status colours.') }}

{{ _('We have not yet engaged a third-party auditor; the assessment is a self-evaluation against the WCAG 2.2 Level AA success criteria.') }}

{{ _('Self-hosted considerations') }}

{{ _('onChange by Sairo is a self-hosted application. Operators can change the deployment in ways that affect accessibility — for example, by injecting custom CSS, embedding the app inside an inaccessible host page, or running an outdated build. The conformance claim above describes the upstream codebase as published. Operators are responsible for the accessibility of their own deployments, and we recommend re-testing after any local customisation.') }}
